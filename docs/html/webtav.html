class webtav.webfront.XSRFHandler(application, request, **kwargs)[source]¶

Bases: webtav.webbase.XSRFBaseHandler

The XSRF handler returns a xsrf token

This handler’s output is included into the header via nginx SSI. By importing via SSI, nginx can cache the output from other handlers.

Otherwise, each request from a non-logged in user would do a full page-load, since the Set-Cookie would bypass the cache.

We do need a separate token for each request, since the site contains a login form. Combining them in nginx lets us have a separate xsrf value while still caching on sessionid.

This inherits from XSRFBaseHandler so that it bypasses the regular login/session path.
